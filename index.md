# ビジネスメール詐欺（BEC）

>>>

### ビジネスメール詐欺（BEC）

ビジネスメール詐欺（Business E-mail Compromise：BEC）とは、巧妙な騙しの手口を駆使した、偽の電子メールを組織・企業に送り付け、
従業員を騙して送金取引に係る資金を詐取するといった、金銭的な被害をもたらすサイバー攻撃です。詐欺行為の準備として、企業内の従業員
などの情報が狙われたり、情報を窃取するウイルスが悪用されることもあります。  
【IPA周知文章から引用】https://www.ipa.go.jp/files/000058478.pdf

>>>

### ビジネスメール詐欺（BEC）

#### 代表的な手順

1. 事前に標的に関する情報を盗む
  - フィッシング、キーロガー等で標的の認証情報を盗みだし、過去のメール等から情報を集める
2. 集めた情報からメールを作成して標的に送り付ける
  - 業務内容等から実際に送付されてもおかしくないような内容で、標的から口座に入金させる。

# 多段階認証

>>>

### 多段階認証

- 多要素認証と多段階認証

>>>

### 認証で使われる要素

- 知識情報
  - パスワード、PINコード、秘密の質問
- 所持情報
  - セキュリティーキー、スマートフォン、スマートウォッチ
- 生体情報
  - 指紋、虹彩（目んたま）、静脈、顔

>>>

### 多段階認証と多要素認証の違い

#### ２段階認証

パスワード + 秘密の質問

#### ２要素認証

パスワード + SMS認証  
セキュリティキー + 指紋

# WPA3

>>>

## WPA3

Wi-Fiアライアンスが発表した、無線LANのセキュリティを強固にした新しいプロトコルです。  
WPA2の脆弱性「KRACKs」に対応する形で、セキュリティが強化されたWPA3が発表されました。

>>>

## WPA3の特徴

### SAEハンドシェイクによる防護

- KRACKsの対策がされたSAE（Simultaneous Authentication of Equals）ハンドシェイクが導入
  - WPA2では複合化されてしまう暗号化された通信内容を解読不可能にする
- 辞書攻撃・総当たり攻撃からの防護
- 192ビットの暗号化システムCNSAを実装（WPA3-Enterprise）

>>>

## WPA3の脆弱性

SAEハンドシェイクの別名である「Dragonfly」の脆弱性「Dragonblood」【対応済み】
